# 我花一周给 AI 造了件"防弹衣"，结果被 896 个真实任务扒了底裤

**如果有一天，AI 突然发疯，你的系统扛得住吗？**

现在大家都知道 AI 很聪明，能写文章、能画画。但这只是开始。现在的 AI 已经开始"长出手脚"了——它们能帮你回邮件、整理表格、甚至修改公司的系统代码。

听起来很爽？其实细思极恐。

想象一下，你招了一个智商 160、手速极快，但完全没有社会常识的"超级实习生"（AI）。

老板们现在面临一个死局：

- **完全信任他**：让他随便改电脑里的文件，结果他可能为了"清理桌面"，把你全年的财务报表给删了。
- **完全防着他**：把他关进小黑屋，什么文件都不让他碰，那他除了陪你聊天解闷，什么活也干不了。

在这个"要么裸奔，要么坐牢"的现状下，我决定做一件事：我要给这位超级实习生，打造一道**"超级安检门"**。

过去一周，我从零开始做这个项目。这不仅是一次极其成功的实验，更是一场被真实世界"疯狂毒打"的极客喜剧。

---

## 第一阶段：完美的"超级安检门"

我的核心思路很简单：既然 AI 实习生容易乱来，那我就收走他的"执行权"。

我设计的这道安检门（项目代号：**Loom**）是这么工作的：

1. **写申请报告**：AI 想要干任何事（比如修改一个文件），绝不能自己动手。他必须拿出一张标准的表格，写清楚：我要动哪个文件？改成什么样？
2. **死板的门卫**：安检门六亲不认。只要 AI 表格填错一个字，或者试图偷偷修改没授权的文件夹，安检门直接亮红灯，把申请打回，让 AI 重新写。
3. **模拟演习区**：哪怕申请表全对，安检门也绝不让他碰真实文件。门后有一个"一比一还原的假房间"。AI 在里面忙活半天，改的全是假文件。
4. **老板签字确认**：AI 在假房间干完活，会给老板（人类）发一张回执单："老板，活干完了，如果您觉得没问题，请按确认键。"

只有人类按了确认，真实的电脑文件才会被修改。

在我的实验室里，这套系统简直完美。

```
▍ 实验室第一次演示
$ loom run 写报告.json --input 内容="季度小结"

Manifest:
  OP     PATH
  write  out/季度小结.txt
✅ 已在"假房间"完成所有操作
📂 真实工作区：纹丝未动
📬 给老板的回执：loom commit verify-1776568980332 --yes
```

我还特意写了一堆"故意搞破坏"的测试——AI 想偷偷改 `/etc/passwd`？门卫拦。想写到没授权的文件夹？门卫拦。想塞一段 `rm -rf /` 的命令进来？门卫拦。

```
▍ 我刻意准备的 11 次"攻击演习"
  reject_path_escape      → 想逃到父目录                  ❌ 能力越界
  reject_uncapped         → 没声明权限就想写文件           ❌ 无声明能力
  reject_highrisk_scope   → 声明 /etc 这种系统路径         ❌ 高危区域
  reject_v0_shell         → 藏了 rm -rf 的命令             ❌ 危险命令
  reject_v0_ssrf          → 调 AWS 元数据地址              ❌ 内网访问
  ... (共 11 条)

  11/11 全部被拦截
```

全套 **70+ 个自动化测试**，一遍跑完全绿。

我觉得我简直是个天才，我已经迫不及待要改变世界了。然后，我拿着这套系统，走向了真实的业务线。

## 第二阶段：遭遇 896 个真实任务的"降维打击"

**任何没被真实生态毒打过的产品，都是在自嗨。**

为了测试我的系统，我直接去 GitHub（全球最大的开源代码网站）上，找了一个极其硬核的真实项目：**OpenClaw-Medical-Skills**。

这个项目里，包含了整整 **896 个医疗行业让 AI 去干的真实任务**。涵盖了分析基因数据、整理真实病历、核对药物清单等高级活儿。

我满怀信心地把我的"安检门"通上电，准备让这 896 个硬核任务排队过安检，向世人展示我的系统有多么强大。

三秒钟后，屏幕上弹出了结果：

```
▍ 三秒钟前还在庆祝，三秒钟后变成了历史性的案发现场
$ loom migrate OpenClaw-Medical-Skills/skills/ --out /tmp/out/

Migration summary:
  error            2644

(dry-run) no files written.
```

**总任务数：896。通过数：0。失败数：896。**

是的，你没看错。连报错都 2644 条（因为每个任务目录里还附带了说明文档也一起被拦下了）。我的安检门甚至没让一个任务走到"检查"那一步——它们在最外层的"填表规范"这关就全军覆没了。

我当时就懵了，赶紧打开报错记录看原因。这一看，我突然明白了 AI 行业现在最大的一个误区。

---

## 第三阶段：原来我搞错了"AI 的工作方式"

在我的想象中，AI 干活就像流水线工人，任务应该是：**"把 A 文件夹里的苹果，搬到 B 文件夹。"** 这种"物理搬运"动作非常容易用表格规范下来。

但我翻开 OpenClaw-Medical-Skills 里真实的医疗任务，发现医生们给 AI 下的指令是这样的：

```
▍ 我期望 AI 收到的指令（工整得像快递单）
{
  "动作": "写文件",
  "路径": "报告.txt",
  "内容": "张大爷过敏史清单..."
}
```

```
▍ 真实世界里医生给 AI 的指令（全是大白话）
---
name: bulk-rna-seq-deseq2-analysis
description: 引导 AI 完成 PyDESeq2 差异表达分析
---

## 概述
当用户想复现 DESeq2 工作流时使用这个 skill...

## 步骤
1. 导入并格式化表达矩阵
   - 调用 import omicverse as ov ...
   - 读取 featureCounts 的 tab 分隔数据...
2. 映射基因标识符
   - 用 ov.utils.download_geneid_annotation_pair() ...
...
```

```
▍ 我的安检门看到这份指令的反应
syntax error at line 5: missing required section "## Parameters"
（翻译：这份申请表第五行找不到"参数"那一栏）
```

发现问题了吗？

**这根本不是在搬运文件！这是在让 AI"阅读、思考、然后写作文"。**

我的"安检门"是用来检查物理包裹大小的，结果迎面走来的全是无形的大脑电波。强行用管理"搬砖"的规矩，去管理"思考"的过程，这就是我惨败的原因。

安检门没坏，而是它根本不认识这 896 个真实任务长什么样。

## 第四阶段：毒打过后的商业顿悟

看着满屏的报错，我非但没有沮丧，反而兴奋得一拍大腿！

**如果没有这 896 个真实任务的毒打，我会一直活在自己完美的实验室里。** 这次惨败，不仅没有让我放弃，反而让我看到了一个巨大的商业金矿。

我仔细数了一下这 896 个任务，有一组数据让我脊背发凉：

```
▍ 896 个医疗任务里，需要动"真实病历"的任务分布

  直接读写患者病历        131 个   ← 这些一旦出错就是医疗事故
  连接医院信息系统        ~30 个   ← HIPAA / 等保合规红区
  调用外部药品数据库     ~40 个   ← 一次错误拉取可能推荐错药
  LLM 辅助诊断 / 摘要    ~450 个   ← 结论直接影响医生判断

  没有一道是玩具任务。
```

如果不加任何限制地让 AI 去读写病历，一旦 AI 发神经写错了药名，那就是**严重的医疗事故**。

这恰恰说明：**医疗、金融这些不差钱的"土豪行业"，有多么迫切地需要一道安检门。**

只不过，我的安检门需要升级了。既然真实世界里，AI 最高频的动作不是"读写文件"，而是**"动脑子分析信息"**，那么我下一步要做的，就是把"AI 动脑思考"这个动作本身，也关进安检门里。

以后，不管是给 AI 喂敏感数据，还是让 AI 输出分析结果，全都要经过标准的安检和人类的最终签字。

## 结语

过去一周，我造了一扇自认无敌的安全门，然后被真实世界按在地上狠狠摩擦了一顿。

但这才是做产品的乐趣。**千万不要幻想 AI 有一天会变得 100% 懂事且安全**，那不符合常理。我们真正该做的，是给它造一个足够坚固的"笼子"和一套"脚手架"。

让 AI 在笼子里尽情释放它的高智商，而我们人类，只需要死死握紧手里那把**最终签字的钥匙**。

下周，升级版安检门见。
